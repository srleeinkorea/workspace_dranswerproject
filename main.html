PRD — 중증소아 재택의료(인공호흡기 환아 관점)

이 문서는 “재택의료 서비스를 받는 인공호흡기 의존 환아”의 관점에서, 서비스 경험(안전·일상·불필요한 내원 감소)을 달성하기 위한 제품 요구사항(PRD) 초안입니다.
※ 본 문서는 의료 조언/진단이 아니며, 임상 판단/처방은 담당 의료진과 상의가 필요합니다.

1. 제품/서비스 한 줄 정의

환아가 집에서 인공호흡기와 함께 살아가며 “안전”과 “일상”을 유지하도록 돕는 재택의료 연계 서비스/제품.

Safety Continuity Burden↓

서비스 핵심(환아 관점)

  • 갑자기 숨이 불편해질 때도 “어떻게 해야 하는지”가 명확하다
  • 필요한 경우에만 병원에 가고, 불필요한 이동/대기가 줄어든다
  • 보호자가 교육과 상담을 통해 “안전한 돌봄”을 지속할 수 있다

프로토콜 기반 서비스 구성(요약)

  • 관리계획 수립 및 정기 점검
  • 방문 진료/간호/재활 + 교육·상담
  • 전화상담 기반 상시적 환자관리(현실 제약 반영)

2. 배경 및 문제정의(환아 관점)

환아와 가족이 실제로 겪는 “집에서의 돌봄 난이도”와 “병원 이용 부담”을 제품 문제로 정의합니다.

의료공백

야간/휴일/외래 사이 시간에는 상태 변화를 스스로 판단해야 하는 부담이 크다.

기기 의존

인공호흡기/회로/마스크 문제는 불편과 위험을 동시에 유발하므로 빠른 점검 루트가 필요하다.

내원 부담

이동·대기·입원은 환아/가족 모두에 큰 부담이므로 “필요한 내원만” 유도해야 한다.

3. 목표(Goals) / 비목표(Non-goals)

서비스가 달성해야 할 가치와, 의도하지 않는 범위를 명확히 분리합니다.

Goals

  • 안전: 위험 신호를 조기에 인지하고 적절한 대응 경로(가이드/상담/내원)를 제공
  • 지속성: 환아 상태에 맞춘 관리계획이 정기적으로 점검/업데이트됨(월 단위 점검, 연 단위 재평가)
  • 경험: 보호자 교육·상담과 기록을 통해 자기관리 역량 강화
  • 효율: 불필요한 응급실/외래 이용 감소(필요 내원은 놓치지 않음)

Non-goals

  • 응급상황을 대체하지 않음(응급은 119/응급실 우선)
  • 앱/AI가 진단·처방을 자동으로 내리지 않음(Decision Support/연계 중심)
  • 재택의료팀 운영(인력/행정)은 “유저 기능”이 아니라 운영 요구사항으로 분리 관리

4. 사용자 정의(User) — 환아 중심

PRD의 User는 “서비스 가치의 주체”를 기준으로 정의합니다.

구분 정의 제품 접점
Primary User
환아		 서비스가 궁극적으로 안전/일상/삶의 질을 개선해야 하는 대상(인공호흡기 의존 중증소아) 직접 조작이 어렵다면 보호자 UX로 대체(Proxy)
Proxy Operator
보호자		 실제 앱 조작/상담 요청/조치 수행을 담당하는 대리 사용자 App: 알림/가이드/상담/기록
Clinical User
의료진		 의사결정 주체(Decision Support). Level 3 확인 및 개입/설정 조정 책임(HITL) Dashboard: 우선순위/케이스/근거/리포트
Provider/Operator
재택의료팀		 서비스 제공 주체(운영/방문/상담/회의). 유저가 아니라 운영 요구사항으로 분리 운영 콘솔(옵션), 업무 프로토콜
Intended User(규제 관점): 보호자(Proxy)와 의료진(Clinical User)을 명확히 구분하고, 진단/처방이 아닌 의사결정 지원으로 설계합니다.

5. 핵심 시나리오(환아 여정 기반)

재택의료 프로토콜 흐름을 제품 여정으로 번역합니다.

온보딩(등록) & 초기평가

  • 대상자 등록 + 보호자 동의 + 비상연락망 확보
  • 재택의료 관리계획서 생성(기본정보/최근 의료이용/현재 서비스/교육 환경 등)
  • 기기(인공호흡기) 사용 현황 및 위험 요소 점검

정기 방문 + 교육·상담

  • 방문 진료/간호/재활(처방 범위) 제공
  • 보호자 대상 교육·상담 제공 및 기록
  • 반복 문제(예: 회로 이탈/누출) 예방 중심으로 재교육

상시적 환자관리(상담/모니터링)

  • 정규시간 전화상담 기반 요청 대응(현실적 운영 제약 반영)
  • 최소 월 1회 관리계획 점검/모니터링 및 기록
  • 악화 의심 시 즉시 내원/의료진 연계

재평가/연장/퇴록

  • 연 1회 재평가(초기평가 수준) 및 연장/퇴록 결정
  • 필요 시 팀 논의 기반 계획 재수립
  • 서비스 불필요/원치 않음/안전상 불가 등 퇴록 조건 관리

6. User Stories (환아 관점, 보호자 Proxy 실행)

환아 가치(안전·불안 감소·일상 유지)를 중심으로 작성하고, 실제 조작은 보호자가 수행합니다.

Patient Value Proxy UX Safety-first
US-P-01

악화가 의심될 때, 즉시 올바른 대응 경로로 연결되고 싶다

Emergency Routing
As a
인공호흡기 의존 환아(가치 주체)
I want
숨이 불편해지거나 상태가 급격히 변할 때, 보호자가 즉시 상담/내원/응급 루트를 찾을 수 있길 원한다.
So that
위험을 줄이고 빠르게 안정될 수 있다.

Acceptance Criteria

  1. 위험 수준(Level)에 따라 “관찰/외래/즉시 응급”이 한 문장으로 제시된다
  2. 응급(Level 3)일 때는 안내가 최상단에 고정되고 추가 클릭 없이 연락/가이드로 이동된다
  3. 모든 안내에는 “진단/처방이 아님” 및 “의료진/응급실 우선” 안전 문구가 포함된다
US-P-02

기기(회로/마스크) 문제가 생기면 단계별 점검이 쉽고 빠르길 원한다

Device Checklist
As a
인공호흡기 의존 환아
I want
회로 이탈/누출/마스크 불편 등 기기 문제가 의심될 때, 보호자가 바로 점검 순서를 안내받길 원한다.
So that
불편과 위험을 동시에 줄이고 야간 불안을 낮출 수 있다.

Acceptance Criteria

  1. “점검 1→2→3단계(체크리스트)” 형태로 제공된다
  2. 각 단계에 “정상/비정상 판단 기준”이 짧게 포함된다
  3. 반복 발생 시 “외래 점검/방문 점검 권고”가 자동 노출된다
US-P-03

병원에 자주 가지 않아도 집에서 정기적으로 상태 확인과 재활을 받을 수 있길 원한다

Continuity Visits
As a
인공호흡기 의존 환아
I want
정기 방문(진료/간호/재활)과 교육이 집에서 이어지길 원한다.
So that
학교/일상을 더 유지하고 불필요한 이동을 줄일 수 있다.

Acceptance Criteria

  1. 방문 일정(진료/간호/재활)이 달력 형태로 공유된다
  2. 방문 후 “오늘 한 것(평가/교육/조치)”이 3줄 요약으로 남는다
  3. 처방/진료 변경이 필요한 경우 의료진 확인 플로우로 연결된다
US-P-04

내 상태 변화에 따라 관리계획이 업데이트되길 원한다

Plan Review
As a
인공호흡기 의존 환아
I want
악화/호전 등 상태 변화가 있을 때 관리계획이 최신화되길 원한다.
So that
불필요한 내원·불안은 줄이고 필요한 개입은 놓치지 않는다.

Acceptance Criteria

  1. 월 1회 이상 “계획 점검/모니터링 완료” 상태가 표시된다
  2. 연 1회 재평가 일정과 결과(연장/퇴록/변경)가 기록된다
  3. 계획 변경 시 보호자에게 “무엇이 바뀌었는지(핵심 3개)”가 공유된다
US-P-05

보호자가 교육·상담을 반복해서 받고 기록이 남아 돌봄이 안정적이길 원한다

Education Record
As a
인공호흡기 의존 환아
I want
보호자가 기기/상태를 이해하도록 교육과 상담이 표준화되고 기록되길 원한다.
So that
보호자의 불안을 줄이고, 내 돌봄이 더 안정적으로 유지된다.

Acceptance Criteria

  1. 교육/상담 항목(예: 기기 점검, 증상 관찰, 비상 대응)이 체크리스트로 관리된다
  2. 교육 기록은 날짜/담당/주제별로 검색된다
  3. 반복 알람/반복 문의가 있으면 관련 교육 콘텐츠를 자동 추천한다

7. 기능 요구사항 (Functional Requirements)

프로토콜의 핵심 축(관리계획·방문·상담·재평가)을 제품 기능으로 구체화합니다.

FR-A. 관리계획(Plan)

  • FR-A1 환자별 관리계획서 생성/조회/수정(기본정보, 최근 의료이용, 서비스 이용 현황 등)
  • FR-A2 월 1회 이상 점검/모니터링 수행 체크 및 기록
  • FR-A3 연 1회 재평가(연장/퇴록/계획 변경) 일정·결과 관리

FR-B. 방문(Visit) + 교육·상담(Education)

  • FR-B1 방문 일정 관리(진료/간호/재활) 및 보호자 공유
  • FR-B2 방문 교육·상담 기록(주제/내용/담당/다음 액션)
  • FR-B3 반복 문제(회로 분리/누출 등) 발생 시 재교육 트리거

FR-C. 상담/연계(Access)

  • FR-C1 정규시간 상담 접점(클릭 투 콜/요청 티켓)
  • FR-C2 상담 요청→응답→결과 기록(누락 방지)
  • FR-C3 위험 수준에 따라 “응급/외래/관찰” 라우팅

FR-D. 의료진(Decision Support)

  • FR-D1 고위험 환자 우선순위(큐) + 근거 표시
  • FR-D2 Level 3 케이스 생성/확인/에스컬레이션(HITL)
  • FR-D3 30일 요약 리포트(사용/환기/압력/누출/알람)
주의: “재택의료팀 운영(인력/회의/행정)”은 User 기능이 아니라 별도의 Operational Requirements로 관리하는 것이 PRD 구조상 안전합니다.

8. 비기능 요구사항 (NFR)

안전·보안·신뢰·사용성을 제품 수준에서 보장하기 위한 조건입니다.

NFR-1 안전(Safety)

  • 응급(Level 3) 안내는 가장 앞에, 가장 적은 클릭으로
  • 진단/처방 유도 금지(안전 문구 + 의료진 상담 권고)
  • Degraded Mode(센서/데이터 불안정) 명확 표시

NFR-2 보안/프라이버시

  • 동의 기반 접근 + 권한(RBAC)
  • 감사 로그(조회/수정/확인/에스컬레이션)
  • PHI 최소화 원칙

NFR-3 신뢰/가용성

  • 월 1회 점검 누락 방지(자동 리마인더)
  • 상담 요청 티켓의 상태 추적(접수/진행/완료)
  • 핵심 화면(응급 라우팅)은 저사양에서도 동작

9. 데이터/평가지표 설계

평가 프레임(구조-과정-결과)을 PRD 성과지표로 정렬합니다. (심사/확산/운영에 유리)

구분 지표 예시(최소) 측정 주기
Structure
역량		 인력 구성, 교육 이수, 자문/운영 체계, 보안/권한 정책 분기/반기
Process
수행		 상담 횟수/목적/형태/소요시간, 방문 수행률, 월 1회 점검 이행률, 교육 수행률 월/분기
Result
성과		 응급실 방문, 입원/재입원, 재원기간, 가족 만족도/삶의 질, 안전 이벤트(Zero harm) 분기/연

인공호흡기 원격 지표(확장 옵션)

본 PRD는 “재택의료 프로토콜 기반”이므로, 기술 확장 시 인공호흡기 데이터(MV/RR/PIP/Leak 등)를 Process/Result 지표의 보조 근거로 연결할 수 있습니다.

  • 예: 반복 회로 분리/누출 → 교육/피팅 점검 성과 측정
  • 예: 사용 시간/패턴 → 자기관리 지속성(프로세스) 보조 지표

운영지표(필수)

  • 월 1회 점검 이행률(누락 0 목표)
  • 상담 응답 시간(접수→1차 응답)
  • Level 3 발생 시 의료진 확인 시간(ACK)
  • 교육/상담 기록 작성률

10. MVP 범위(현실 제약 반영)

운영 현실을 고려해 “정규시간 상담 + 정기 점검/방문 + 기록체계”부터 구현합니다.

Plan Visit Access(9–18)

MVP 포함
