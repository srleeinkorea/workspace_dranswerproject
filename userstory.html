본 섹션은 PRD에 포함되는 User Story + Acceptance Criteria입니다. Intended Users는 보호자(Caregiver)와 의료진(Clinician)으로 한정합니다.

1. Intended Users 정의

SaMD/심사 관점에서 “누가 사용하는가”를 명확히 정의합니다.

Primary User 1: 보호자 (Caregiver)

  • 가정 내 실행 주체 (조치 수행)
  • 의료 의사결정 권한 없음 (진단/처방 불가)
  • 모바일 앱 기반 인터페이스 사용

Primary User 2: 의료진 (Clinician)

  • 의사결정 주체 (Decision Support User)
  • Dashboard 기반 케이스 확인 및 개입
  • Human-in-the-loop 책임자 (Level 3 확인/에스컬레이션)
Note: 재택의료팀은 User가 아니라 운영/워크플로우 소유자(Operator/Stakeholder)이며, 별도의 Operational Requirements로 관리합니다.

2. 보호자 User Stories

EPIC C1 — 위험 인지 및 대응(알림 → 가이드 → 품질관리 → 상담 판단)

App Safety-first
US-C1-01

실시간 위험 알림 수신

Level 3 SLA
As a
보호자
I want
위험 상황 발생 시 즉시 알림을 받고
So that
아이의 상태 악화를 지연 없이 인지할 수 있다.

Acceptance Criteria

  1. Level 3 이벤트 발생 시 5초 이내 푸시 알림 발송
  2. 알림에 포함: 이벤트 유형(예: SpO₂ 저하/Low TV/회로 분리), 발생 시각, 위험 단계(Level)
  3. 알림 클릭 시 해당 이벤트 상세 화면으로 이동
US-C1-02

단계별 행동 가이드 제공

Checklist HITL
As a
보호자
I want
위험 단계에 맞는 행동 지침을 단계별로 안내받고
So that
표준 절차에 따라 대응할 수 있다.

Acceptance Criteria

  1. Level 1~3별 가이드 분기
  2. 단계별 체크리스트 UI 제공(1→2→3단계)
  3. 각 단계에 “다음 단계로 넘어가는 조건” 명시
  4. Level 3의 경우 “의료진 자동 연락 진행 중” 상태 표시
US-C1-03

센서 이상 및 미착용 감지

Degraded Quality
As a
보호자
I want
센서 미착용 또는 신호 오류를 즉시 안내받고
So that
데이터 누락 없이 안전을 유지할 수 있다.

Acceptance Criteria

  1. SpO₂ 신호 품질 저하 감지 시 알림 발생
  2. 재부착 가이드 UI 제공
  3. 정상 신호 복구 시 “정상” 상태 표시
  4. 센서 불안정 시 Degraded Mode 자동 전환 안내
US-C1-04

상담 필요 여부 명확화

Safety
As a
보호자
I want
지금 병원에 연락해야 하는지 명확히 알 수 있고
So that
과소 대응이나 과잉 대응을 줄인다.

Acceptance Criteria

  1. Level 3는 “즉시 응급실 방문”을 명확히 표시
  2. Level 2는 “외래 예약 권장”을 명확히 표시
  3. 모호한 상황 질문/입력 시 “안전 우선” 안내가 우선 노출
US-C1-05

신뢰 가능한 Q&A 응답

RAG Guardrail
As a
보호자
I want
질문에 대해 출처가 있는 답변을 받고
So that
잘못된 정보에 기반한 판단을 하지 않도록 한다.

Acceptance Criteria

  1. 답변 하단에 출처(문서/프로토콜/가이드) 표시
  2. 응급성 질문은 Triage 우선 처리(안내/알림)로 전환
  3. 진단/처방 요구 시 의료 상담 권고 문구 자동 삽입

3. 의료진 User Stories

EPIC CL1 — 임상 의사결정 지원(우선순위 → 케이스 생성 → 근거 확인 → 임계값 조정 → 요약 리포트)

Dashboard Decision Support
US-CL1-01

고위험 환자 우선순위 확인

Queue Explain
As a
의료진
I want
위험 환자가 자동 정렬된 대시보드를 보고
So that
진료 및 개입 우선순위를 효율적으로 결정한다.

Acceptance Criteria

  1. 환자 리스트가 위험점수 기준으로 자동 정렬된다
  2. 각 환자 카드에 위험 근거 Top 3가 표시된다
  3. Level 3 환자는 리스트 상단에 고정된다
US-CL1-02

Level 3 자동 케이스 생성

HITL Audit
As a
의료진
I want
응급 이벤트 발생 시 자동으로 케이스가 생성되고 알림을 받고
So that
개입 지연을 방지한다.

Acceptance Criteria

  1. Level 3 발생 시 자동 케이스가 생성된다
  2. 의료진에게 알림이 발송된다
  3. 확인(ack) 전까지 “미확인” 상태가 유지된다
  4. 설정된 시간이 경과하면 자동 에스컬레이션된다
US-CL1-03

트리아지 판정 근거 확인

Explainable
As a
의료진
I want
시스템이 왜 해당 Level을 판정했는지 확인하고
So that
임상적으로 설명 가능한 결정을 한다.

Acceptance Criteria

  1. 판정 근거 지표가 표시된다(예: SpO₂ 82%, RR 38 등)
  2. Rule 기반/ML 기반 여부가 구분되어 표시된다
  3. 판정 로그를 기록/조회할 수 있다
US-CL1-04

환자별 임계값 조정

Config Versioning
As a
의료진
I want
환자 특성에 따라 임계값을 조정하고
So that
과잉 알람과 누락을 최소화한다.

Acceptance Criteria

  1. 환자별 프로파일 설정이 가능하다
  2. 임계값 변경 시 버전이 기록된다
  3. 변경 이력을 조회할 수 있다
US-CL1-05

30일 요약 리포트 확인

Report Export
As a
의료진
I want
외래 전에 30일 요약 데이터를 확인하고
So that
진료 효율을 높인다.

Acceptance Criteria

  1. 사용시간, 환기, 압력, 누출, 알람 요약이 포함된다
  2. 추세 그래프가 제공된다
  3. PDF 다운로드(또는 내보내기)가 가능하다

4. Non-User Functional Guardrails (PRD 명시)

User Story와 분리하여, 안전/책임/감사 가능한 운영을 위해 반드시 지켜야 하는 시스템 제약을 기술합니다.

안전/책임 구조

  • Level 3는 보호자 취소 불가(안전 우선)
  • Level 3 발생 시 의료진 확인(HITL) 강제
  • 진단/처방 요청에 대한 안전 응답(면책/상담 권고)

감사/보안

  • 모든 판정/확인/에스컬레이션은 감사 로그 기록
  • PHI 접근은 RBAC(권한 기반) 적용
  • Degraded Mode 전환 조건 및 안내 문구 명시
면책: 본 문서는 제품 요구사항(PRD) 목적이며 의료 조언/진단이 아닙니다. 임계값 및 중재 기준은 담당 의료진과 협의하여 설정해야 합니다.
